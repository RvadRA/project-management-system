# Политика хранения данных - Система управления проектами

## 1. Типы хранимых данных

### 1.1. Данные пользователей

- Профили пользователей (имя, email, контактная информация)
- Аутентификационные данные (хэши паролей)
- Роли и права доступа
- История активности пользователей

### 1.2. Проектные данные

- Описания проектов и задач
- Структуры работ (WBS)
- Планы и сроки выполнения
- Бюджетная информация

### 1.3. Операционные данные

- Журналы аудита действий пользователей
- Логи выполнения задач и процессов
- Уведомления и коммуникации
- Метрики производительности

### 1.4. Архивные данные

- Завершенные проекты
- Исторические отчеты
- Архивные копии документов

## 2. Срок хранения данных

| Тип данных           | Срок хранения          | Условия удаления                               |
| -------------------- | ---------------------- | ---------------------------------------------- |
| Данные пользователей | Бессрочно              | По запросу пользователя или нарушению политики |
| Профили сотрудников  | Бессрочно              | Увольнение сотрудника                          |
| Активные проекты     | До завершения + 3 года | Автоматическое архивирование                   |
| Завершенные проекты  | 7 лет                  | Перевод в холодное хранилище                   |
| Задачи и подзадачи   | 5 лет после завершения | Агрегация в отчеты                             |
| Журналы аудита       | 5 лет                  | Компрессия и архивирование                     |
| Уведомления          | 1 год                  | Автоматическое удаление                        |
| Логи системы         | 2 года                 | Ротация по размеру                             |
| Резервные копии      | 30 дней                | Перезапись новыми                              |
| Временные файлы      | 7 дней                 | Автоматическое удаление                        |

## 3. Архивирование данных

### 3.1. Процесс архивирования

- **Ежеквартальное архивирование** завершенных проектов старше 1 года
- **Автоматическое сжатие** журналов аудита старше 1 года
- **Перевод в архив** неактивных данных по расписанию

### 3.2. Хранение архивов

- **Горячее хранилище**: Активные архивы (доступ < 1 мин)
- **Теплое хранилище**: Архивы 1-3 лет (доступ < 5 мин)
- **Холодное хранилище**: Архивы > 3 лет (доступ < 24 часа)

### 3.3. Восстановление из архива

- Запрос через администратора системы
- Восстановление в течение 24 часов для горячих архивов
- Восстановление в течение 72 часов для холодных архивов

## 4. Безопасность данных

### 4.1. Аутентификация и авторизация

- Хранение паролей в виде bcrypt хэшей
- Двухфакторная аутентификация для администраторов
- Ролевая модель доступа (RBAC)
- Принцип наименьших привилегий

### 4.2. Шифрование данных

- Шифрование данных в транзите (TLS 1.3)
- Шифрование чувствительных данных в БД
- Шифрование резервных копий

### 4.3. Контроль доступа

- Аудит всех операций с данными
- Мониторинг подозрительной активности
- Блокировка учетных записей после неудачных попыток входа

## 5. Резервное копирование

### 5.1. Стратегия резервного копирования

- **Ежедневное полное резервное копирование** базы данных
- **Ежечасное инкрементальное копирование** критических данных
- **Еженедельное резервное копирование** файловых вложений

### 5.2. Хранение резервных копий

- **Локальное хранение**: 7 ежедневных копий
- **Удаленное хранение**: 30 ежедневных копий
- **Долгосрочное хранение**: Ежемесячные копии за 1 год

### 5.3. Тестирование восстановления

- **Ежемесячное тестирование** восстановления из резервных копий
- **Документирование процедур** восстановления
- **Обучение персонала** процедурам восстановления

## 6. Удаление данных

### 6.1. Безопасное удаление

- Перезапись удаляемых файлов
- Очистка индексов поиска
- Удаление из всех резервных копий

### 6.2. Аудит удаления

- Логирование всех операций удаления
- Уведомление заинтересованных сторон
- Документирование причин удаления

## 7. Соответствие требованиям

### 7.1. GDPR (Общий регламент по защите данных)

- Право на удаление персональных данных
- Минимизация собираемых данных
- Прозрачность обработки данных

### 7.2. Локальные требования

- Соблюдение корпоративных политик безопасности
- Регулярные аудиты соответствия
- Документирование инцидентов безопасности

## 8. Мониторинг и аудит

### 8.1. Мониторинг использования

- Отслеживание объема хранимых данных
- Мониторинг производительности хранилища
- Предупреждения о приближении лимитов

### 8.2. Регулярные аудиты

- **Ежеквартальный аудит** политик хранения
- **Ежегодный аудит** безопасности данных
- **Проверка соответствия** нормативным требованиям
