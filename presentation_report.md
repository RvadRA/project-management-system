**Дата:** 8 мая 2026 г.
**Версия:** 1.0 (MVP+)
**Статус:** Готовность к демонстрации 95%

---

## 1. Соответствие ключевым целям проекта

### 1.1. Переиспользование блоков планирования (WBS)
*   **Реализовано:** Импорт структуры работ (WBS) из активных/завершенных проектов или из глобального каталога шаблонов.
*   **Мастер переноса:** Автоматическая адаптация сроков, масштабирование трудозатрат (scaling factor) и пересчет дат с учетом рабочих календарей.
*   **Каталог:** Создана база «пакетов планирования» с метаданными по доменам, сложности и средним отклонениям.

### 1.2. Интеллектуальный подбор сотрудников (Matching)
*   **Профили:** Учет навыков (1-5 уровни), сертификатов, опыта в доменах и истории прошлых проектов.
*   **Скоринг:** Реализована формула: `Score = w1*Comp + w2*Exp + w3*Avail + w4*Perf + w5*Cost`. Веса настраиваются.
*   **Конфликт-чекер:** Система предупреждает о перегрузке (>100%), пересечении отпусков и несовпадении ролей.
*   **Сценарии «Что-если»:** В API реализована симуляция влияния сдвига задач на общие сроки проекта.

### 1.3. Бизнес-процессы по шаблонам
*   **Движок процессов:** Поддержка параллельных ветвей и зависимостей между задачами. Завершение задачи автоматически активирует следующую.
*   **Типы задач:** Поддерживаются текстовые отчеты, загрузка файлов, чек-листы и задачи с обязательным утверждением (Approval).
*   **Интеграции:** Заложен фундамент для внешних вызовов (веб-хуки) в рамках этапов процесса.

### 1.4. Единое пространство и Личный кабинет
*   **Интерфейс:** Сотрудник видит приоритетные задачи с SLA. Реализована интерактивная форма сдачи отчетности.
*   **Уведомления:** Система рассылает алерты через Email и Telegram при назначении задач или приближении дедлайнов.

## 2. Технический стек
*   **Backend:** Python 3.10+, Django, Django REST Framework (DRF), PostgreSQL.
*   **Асинхронность:** Celery + Redis (уведомления, проверка SLA, тяжелые пересчеты WBS).
*   **Frontend:** React 18, TypeScript, Vite, Tailwind CSS.
*   **Архитектура:** Fat Models, Service Layer для бизнес-логики (матчинг, импорт планов).

## 5. Инструкция для демо (27-29 апреля)

### Запуск инфраструктуры
```bash
# 1. База данных (Docker)
docker-compose up -d db

# 2. Backend (в venv)
python manage.py migrate
python manage.py runserver

# 3. Очереди задач (Celery)
celery -A config worker -l info -P eventlet
celery -A config beat -l info

# 4. Frontend
cd frontend && npm run dev
```

1. **Каталог шаблонов**: Показать готовую структуру работ для типичного IT-проекта.
2. **Создание проекта**: Импорт шаблона, масштабирование сроков на 1.5x.
3. **Матчинг**: Автоматический подбор команды. Выбор лучшего кандидата на основе Score.
4. **Workflow**: Запуск процесса разработки, прохождение по этапам (TODO -> IN_PROGRESS -> DONE), проверка синхронизации с WBS.
5. **Дашборд**: Демонстрация графиков выработки и загрузки команды.
